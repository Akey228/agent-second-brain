Собрал все идеи по автоматизации и улучшению Obsidian, которые обсуждались в разных сессиях. Плюс мои дополнения в конце.

<b>Структура папок и организация</b>

1. Создать отдельную папку для ежедневных заметок (дневник). Сейчас дневные заметки лежат в корне vault — лучше собрать в отдельную папку типа "Daily" или пронумерованную.

2. Привести все папки к единой нумерации — чтобы была логическая иерархия и порядок.

3. Inbox-воркфлоу: в течение недели все новые заметки падают в Inbox. Каждое воскресенье — разбор: раскидать заметки по нужным местам. После разбора Inbox пустой.

4. Убрать ненужные подпапки (Fleeting Notes, Literature Notes, деление Resources/Areas) — они не используются. Заметки находятся через Home/MOC, поиск или через ассистента, а не через папки.

<b>Домашняя страница и навигация</b>

5. Автоматизировать отображение MOC на домашней странице — через DataView/DataView.js. Чтобы при создании нового MOC он автоматически появлялся на Home, без ручного добавления.

6. Тематическая группировка MOC по колонкам. Например:
   - Hard Editing (технические навыки): After Effects, Premiere Pro, монтаж, ритм, приёмы
   - Soft Editing (бизнес/клиенты): работа с клиентами, портфолио, поиск клиентов
   - Когда набирается 5+ MOC на одну тему — создаётся новая колонка

<b>Файлы и вложения</b>

7. Все изображения, видео и вложения должны идти строго в vault/0. System/Files/ — не создавать новые папки для attachments.

<b>Рабочие заметки</b>

8. Исправить именование Work Daily заметок — добавить пробел между "Work" и "Daily" для консистентности.

<b>Клиентские заметки</b>

9. Ввести клиентские заметки-карточки. На каждого клиента (например, Motion Pulse) — отдельная заметка, куда накапливается вся информация: правила по стилю, идеи, фидбеки, контакты. См. [[Клиентские заметки в Obsidian]]

<b>Еженедельный обзор</b>

10. Создать повторяющуюся задачу в Todoist на каждое воскресенье: "Перебрать все заметки в Obsidian" — разбор Inbox, проверка актуальности заметок.

<b>Система редакторских референсов</b>

11. Создать дашборд/доску с монтажными референсами:
   - Проблема: при начале монтажа непонятно какие приёмы/стили применить
   - Решение: коллекция примеров видео по темам — как показать течение времени, как сделать переход, цветокоррекция и т.д.
   - На каждый приём — несколько примеров (5-10 видео)
   - Можно листать как доску вдохновения, когда застрял
   - Нужно продумать: как хранить видео-референсы (ссылки? скачанные файлы?), как категоризировать по темам

12. Расширить текстовые заметки по монтажу — сторителлинг, эффект Кулешова, ритм, как улучшить пейсинг. Уже частично есть, но нужно развивать.

<b>Плагины (обсуждались ранее)</b>

13. Editing Toolbar — удобное меню форматирования для Markdown
14. Iconize — иконки для папок
15. Self-hosted LiveSync — синхронизация через свой сервер
16. Various Complements — автоподстановка ссылок на заметки

<b>Мои дополнения</b>

17. Шаблоны заметок (Templates) — настроить шаблоны для частых типов заметок: клиентская карточка, MOC, ежедневная заметка, рабочая заметка. Через Templater плагин — одно нажатие и шаблон готов.

18. Периодические заметки — помимо дневника, можно делать еженедельный и ежемесячный обзор. Раз в неделю — краткий итог, что сделано, какие паттерны заметил. Раз в месяц — более глубокий анализ прогресса.

19. Быстрый поиск по тегам — настроить систему тегов так, чтобы можно было быстро найти все заметки по теме (например, все заметки с тегом "монтаж" или "клиент").

20. Canvas для визуального планирования — встроенная функция Obsidian Canvas может быть полезна для раскадровки, планирования видеопроектов и визуализации связей между идеями.
